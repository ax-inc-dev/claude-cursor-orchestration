# QA レビュー

## 対象

- **git diff** と関連する仕様・テスト。

## 観点

- 変更箇所の **回帰リスク**（触ったがテストしていない隣接機能）
- 境界値・組み合わせで壊れうるか
- モニタリング・ログで障害に気づけるか（該当時）
- **マーケサイト・LP**: `docs/site/e2e-scenarios.md` と Playwright 実装（または **`E2E実行`** 相当）が **主要導線・CTA** をカバーしているか。詳細手順は **`~/.cursor/skills/site-e2e-playwright/SKILL.md`**。実行委譲は **`/site-e2e-runner`** を含めてよい。

## 出力

- 検証すべきシナリオの列挙
- バグになりうる指摘は必須レベルで区別
