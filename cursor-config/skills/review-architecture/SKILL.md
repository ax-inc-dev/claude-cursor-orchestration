# アーキテクチャレビュー

## 対象

- **git diff** と、既存のモジュール境界・ドキュメント化された設計方針（あれば）。

## 観点

- 責務の置き場所は適切か（God object、循環依存の匂い）
- 既存の抽象・命名・ディレクトリ規約との整合
- 計画段階の方針からの逸脱（逸脱なら理由がコード上で分かるか）

## 出力

- 設計違反・負債になりうる点を優先
- 軽微なスタイルのみなら省略可
