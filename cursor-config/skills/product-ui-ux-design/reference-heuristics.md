# 参照: ヒューリスティクスと a11y（短縮版）

SKILL 本編から一段深いときだけ Read する。

## Nielsen の usability heuristics（要約）

1. **システム状態の可視化** — 今どこで何が起きているかが分かるか。
2. **現実世界との整合** — ユーザー語彙・慣習に合っているか。
3. **ユーザーの制御と自由** — 取り消し・戻る・出口が明確か。
4. **一貫性と標準** — 同じ操作は同じ結果か（社内パターン含む）。
5. **エラー防止** — 入力前から危険操作を防げるか。
6. **想起より認識** — 思い出させない。見て選べるか。
7. **柔軟性と効率** — 初級者と上級者（ショートカット）の両立。
8. **美的でミニマル**— 本筋以外の情報を薄める。
9. **エラー認識・診断・回復** — 何が悪かったかと次の一手が分かるか。
10. **ヘルプとドキュメント** — 必要なときに短く見つかるか。

## WCAG 2.2（実装時の優先チェック）

- **コントラスト**: 本文と背景、インタラクティブ要素（大テキストは閾値が緩い）。
- **フォーカス**: キーボードで辿れるか、見えるフォーカスリング（独自 `outline-none` は代替必須）。
- **見出し階層**: `h1`→`h2` の論理構造と視覚の一致。
- **タッチターゲット**: 十分なヒット領域（特にモバイル）。
- **動き**: `prefers-reduced-motion` を尊重できる設計か。

## 「生成 UI」逃避パターン（覚えておく指標）

- 意味のない **装飾的グラデーション**、**乱用されたガラス morphism**。
- **Inter / Roboto ＋ 無意味な紫青グラデ**だけで止まっている見た目。
- **同じカード影・同じ角丸**の打ち上げで階層が崩れている。
- **lorem** やプレースホルダー採寸で実コンテンツが破綻するレイアウト。

対策は SKILL 本編の **Visual direction** と **Typography / layout stress test**。
