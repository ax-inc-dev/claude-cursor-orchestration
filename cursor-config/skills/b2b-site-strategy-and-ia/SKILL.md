# B2B Site Strategy & IA

## When to Use

- [site-build-orchestrator](../site-build-orchestrator/SKILL.md) の Phase 1
- コピーやページ構成がブランド固有でない「generic LP」になっていると感じるとき（再設計）

## 手順

入力として **事業の事実**（サービス内容、実績数字、事例、競合との違い、既存顧客の声）をユーザーから集める。足りなければ質問して埋める。

以下 6 設計物を **この順で** 出力し、**作業中リポの** `docs/site/PRD.md` と `docs/site/ia-decision-map.md` に転記可能な Markdown で書く。

### 1. 価値提案シート

| 項目 | 記載内容 |
|------|----------|
| 誰向け | ペルソナまたはセグメント（役職・業種・粒度は案件に合わせる） |
| いまの悩み | 具体的な業務・成果・リスクの言語化 |
| 放置するとどうなるか | 時間・コスト・機会損失など |
| なぜ自社か | 他手段・競合と差がつく一点 |
| 導入後どう変わるか | 観測可能な変化（数字・頻度・負荷） |
| 一言要約 | 「◯◯な企業向けに △△ を ██ で届ける」形式推奨 |

### 2. ターゲット別メッセージマップ

意思決定に関わる人物ごとに表形式：

| ステークホルダー | 主な関心事 | 刺さる論拠（事実） | 避ける表現 | 推奨 CTA 温度感 |
|------------------|------------|-------------------|------------|-----------------|

### 3. カスタマージャーニー別導線

状態の例（案件で言い換えてよい）：

- 課題がまだ曖昧
- 比較・検討
- 導入直前

各状態について：**必要な情報** / **読ませたい順** / **次に進めるアクション** を表にする。

### 4. 意思決定マップ（サイトマップより優先）

ページまたは主要セクション単位で：

| ID | 名前 | 存在理由（目的） | 読了後に持ってほしい信念 | 次の一手（CTA） |
|----|------|------------------|--------------------------|-----------------|

### 5. 信頼要素の棚卸し

実績・数字・ロゴ・事例・認証・メディア・顔出し・比較表・方法論・失敗パターンへの言及などを列挙し、**各要素の証拠レベル**（一次情報 / 抽象）をメモする。無い場合は「取得が必要」と明示する。

### 6. コンテンツトーンと禁止表現

- **トーン**: 例）具体数字優先・伴走・再現性・安っぽい煽りはしない
- **禁止語リスト**: 例）革新的、最先端、包括的に支援、課題解決します、DX を加速、など案件で増やす
- **推奨置換ルール**: 抽象語 → 具体語・数値・プロセス名

## 品質チェック（出力直前に自己確認）

- 「3 秒で何の会社か」が価値提案だけで答えられるか
- 各 CTA が **次の意思決定** に接続しているか
- 信頼要素が **ページ/セクションに割り当て可能** か
