# フロントエンドレビュー

## 観点

- **体験設計の先行**: 変更が **UI/ユーザーフローに効く**のに、対応する **`docs/ux-design/`**・**`docs/site/`**・**`DESIGN.md`**・Figma/仕様の更新が無い／コードだけ先に増えている場合は、**仕様プロセス違反**として **Medium 以上**で指摘し、先に `user-first-product-design` / `product-ui-ux-design` / `site-build-orchestrator` 側を整えるよう促す。
- コンポーネント分割・再利用・props の明確さ
- a11y（フォーカス、ラベル、キーボード）
- 不要な再レンダリングやデータ取得の重複
- 既存デザインシステム・CSS規約との整合（プロジェクトに準拠）

## 出力

他のレビューと重複する場合は統合し、FE特有の指摘を明記。
