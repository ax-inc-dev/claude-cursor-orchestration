# アプリ／プロダクトレビュー

## 対象

- 今回の **git diff** または説明された変更範囲。

## 観点

- 仕様・受け入れ条件を満たすか
- エラー時・空状態・権限の挙動
- 文言、導線、一貫性（既存画面との違和感）
- パフォーマンス上問題になりそうな利用パターン

## 出力

- **必須**: 修正必須（ブロッカー） / 推奨 / 任意
- 軽微すぎる指摘（タイポ1文字のみ等）は省略可
